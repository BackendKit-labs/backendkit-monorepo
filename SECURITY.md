# Security Policy

## Supported Versions

We actively support the latest stable versions of all `@backendkit-labs/*` packages.

Security updates are prioritized for:

* Latest major version
* Latest minor releases under active maintenance

Older versions may not receive security patches.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public GitHub issue.

Instead, report it privately by contacting:

[security@backendkitlabs.dev](mailto:security@backendkitlabs.dev)

Please include:

* Package name
* Affected version
* Detailed description of the issue
* Steps to reproduce
* Potential impact assessment
* Suggested remediation (if available)

---

## Response Process

We aim to:

* Acknowledge reports within 48 hours
* Provide an initial assessment within 5 business days
* Coordinate responsible disclosure before public release

Security reports are handled confidentially and with priority.

Thank you for helping keep BackendKit Labs secure.
